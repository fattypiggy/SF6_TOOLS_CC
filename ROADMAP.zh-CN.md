# SF6CC 项目发展规划

本文档描述 SF6CC 的长期发展方向。

这里记录的是项目愿景，而不是严格的版本计划。

相比于发布日期，我们更关注：

* 用户体验
* 架构质量
* 长期维护
* 社区价值

希望 SF6CC 能够持续发展很多年，而不是做成一个短期项目。

---

# 项目愿景

SF6CC 希望打造一套完整的《街霸6》训练生态。

我们的目标不是简单地增加功能，而是真正降低街霸的学习门槛，让更多玩家能够享受训练、分享经验、共同成长。

未来，希望任何一位新玩家，都可以从下载安装到完成第一套连段训练，在几分钟内完成。

让训练变得简单，而不是复杂。

---

# 我们希望解决的问题

目前街霸社区仍然存在许多问题：

* 连段学习成本高
* 指令术语理解困难
* 新手不知道练什么
* 社区资源分散
* 分享格式不统一
* 教学内容难以维护

SF6CC 希望逐步解决这些问题。

---

# 当前阶段（v0.x）

当前阶段重点不是增加功能，而是打好基础。

包括：

* 完善架构
* 完善 UI
* 完善中文体验
* 建立统一 JSON 格式
* 建立文档体系
* 提高代码质量
* 提高可维护性

只有基础足够稳定，未来才能持续发展。

---

# 第一阶段：完善训练工具（v1.x）

重点建设：

* 连段训练
* Hit Confirm
* Distance Viewer
* Script Manager
* Combo 数据管理
* 多语言支持
* 默认配置优化

让 SF6CC 成为一个真正好用的训练工具。

---

# 第二阶段：连接社区（SF6CM）

SF6CM 是独立项目。

负责：

* 社区
* 网站
* 分享
* 搜索
* 下载
* Metadata
* 用户系统

SF6CC 与 SF6CM 保持完全解耦。

双方仅通过 JSON 进行数据交换。

这样：

即使没有网站，

Lua 依然能够正常运行。

---

# 第三阶段：建立统一数据规范

未来希望建立统一的数据格式。

例如：

Combo

↓

JSON

Metadata

↓

JSON

Character Data

↓

JSON

Exception Rules

↓

JSON

未来希望：

不同社区可以共享同一套 JSON。

而不是重复维护多份数据。

网站可以不同。

数据库可以不同。

但数据格式保持一致。

这样全球玩家都能够共享训练内容。

---

# 第四阶段：智能训练

未来计划探索：

* Replay 分析
* 对局分析
* AI 推荐练习
* AI 连段生成
* AI 学习建议
* 自动训练计划

AI 不代替玩家训练。

而是帮助玩家更高效地训练。

---

# 与 WTT 的关系

SF6CC 并不是 WTT 的竞争项目。

我们的目标是：

能够贡献回 WTT 的能力，尽量贡献回去。

例如：

* 多语言
* UI 优化
* Bug 修复
* 公共组件
* 工具函数

这样所有社区都能够受益。

---

# 长期生态

未来希望形成如下结构：

```text
               WTT Core
                    │
      ┌─────────────┴─────────────┐
      │                           │
   SF6CC                     其它地区增强版
      │                           │
      └─────────────┬─────────────┘
                    │
              统一 JSON 数据规范
                    │
      ┌─────────────┴─────────────┐
      │                           │
   SF6CM 网站                 其它社区网站
```

每个社区都可以拥有自己的：

* 网站
* 数据库
* UI
* 用户体系

但共享同一套数据格式。

真正实现全球内容互通。

如果未来有收入，希望主要来自：

# 最终目标

我们的目标，不是做一个越来越复杂的 Mod。

而是建立一个能够持续维护、持续迭代、持续成长的街霸训练生态。

希望未来：

任何一位街霸玩家，

无论来自哪个国家，

都能够使用统一的数据格式，共享全球社区创造的训练内容。

一个共享的 WTT Core。

一个不断发展的 SF6CC。

一个开放、独立的 SF6CM 社区平台。

共同推动《街霸6》社区不断成长。
