# SF6CC 项目愿景

> **让训练更简单，让知识更容易分享，让社区共同成长。**

---

# 为什么会有 SF6CC？

SF6CC 从来不是为了增加更多功能而诞生。

它存在的意义，是希望让更多玩家能够真正享受《街霸6》的训练过程。

我们相信：

真正阻碍玩家进步的，并不是练习本身。

而是：

* 学不会。
* 找不到。
* 看不懂。
* 不知道练什么。
* 没有人分享。

如果这些问题能够解决，那么训练本身就会变得更加轻松。

因此，SF6CC 希望降低学习门槛，而不是提高工具复杂度。

---

# 我们相信

真正有价值的，不是独占知识。

而是让更多玩家因为这些知识而变强。

我们希望：

优秀的连段能够被分享。

优秀的训练能够被传播。

优秀的经验能够不断沉淀。

最终形成整个社区共同拥有的财富。

---

# 我们希望解决的问题

今天，街霸社区仍然存在很多问题。

例如：

* 连段资料分散在各个平台。
* 视频难以检索和维护。
* 同一个连段被重复整理很多次。
* 新玩家很难找到适合自己的训练内容。
* 不同工具之间的数据无法互通。

这些问题并不是某一个工具能够解决的。

因此，我们希望建立统一的数据规范，让内容能够自由流动，而不是被平台限制。

---

# 我们希望建立什么？

我们希望建立的，不是一个 MOD。

而是一套训练生态。

未来，希望形成如下结构：

```text
                WTT Core
                    │
      ┌─────────────┴─────────────┐
      │                           │
   SF6CC                     其它地区增强版
      │                           │
      └─────────────┬─────────────┘
                    │
             统一 JSON 数据规范
                    │
      ┌─────────────┴─────────────┐
      │                           │
      SF6CM                 其它社区平台
```

WTT 提供全球共享的训练核心。

不同地区可以根据自己的语言、文化和使用习惯，构建各自的社区平台。

每个社区保持独立。

每个社区都可以自由发展。

但所有社区共享同一套开放的数据规范。

---

# 与 WTT 的关系

SF6CC 并不是 WTT 的竞争者。

相反，我们希望：

能够贡献回 WTT 的能力，就尽量贡献回去。

例如：

* 多语言支持
* UI 改进
* 公共组件
* Bug 修复
* 工具函数
* 通用训练能力

这些能力属于所有玩家。

而：

* 社区网站
* 社区运营
* 本地化体验
* 社区特色功能

则可以根据不同地区自由发展。

---

# 与 SF6CM 的关系

SF6CM 是一个独立项目。

它不是 Lua 的一部分。

也不属于 WTT。

它负责：

* 社区
* 搜索
* 下载
* Metadata
* 用户系统
* 分享

Lua 永远不要直接依赖网站。

网站也不要控制 Lua。

双方只通过 JSON 数据进行交互。

这样：

即使网站停止运行，

训练功能依然能够继续使用。

---

# 开放，而不是绑定

我们希望：

不同的网站能够存在。

不同的数据库能够存在。

不同的 UI 能够存在。

不同的社区能够存在。

真正共享的，不应该是网站。

而应该是：

**开放的数据格式。**

因为：

平台可以改变。

技术可以升级。

但玩家创造的知识，不应该因为平台变化而消失。

---

# 永久免费

SF6CC 将永久免费。

不会因为付费而限制训练功能。

如果未来项目获得支持，希望主要来自：

* GitHub Sponsors
* 社区赞助
* 请开发者喝咖啡
* 视频内容
* 社区合作

而不是出售训练工具本身。

---

# 社区共建

SF6CC 欢迎任何形式的贡献。

包括：

* Lua 开发
* UI 优化
* 多语言
* Combo 数据
* Character Data
* Exception Rules
* 文档
* 测试
* Bug 报告

每一位贡献者，都在帮助整个社区成长。

---

# 我们真正希望留下什么？

几年之后，

也许今天的代码已经全部重写。

也许今天的网站已经升级。

也许今天的 UI 已经完全改变。

但我们希望真正留下来的，是这些理念：

* 知识应该共享，而不是封闭。
* 数据应该开放，而不是绑定平台。
* 社区应该合作，而不是重复建设。
* 架构应该可维护，而不是不断堆积功能。
* 能够贡献回 WTT 的能力，就尽量贡献回 WTT。

---

# 最后的愿景

我们希望未来，无论来自哪个国家、使用哪种语言，每一位街霸玩家都能够因为开放、共享的训练生态而受益。

真正被分享的，不只是连段。

而是经验。

是热爱。

也是整个社区共同成长的力量。

---

> **我们不是在开发一个训练 MOD。**

> **我们希望建立一个能够持续维护、开放协作、面向未来的街霸训练生态。**
