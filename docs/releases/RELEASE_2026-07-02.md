# Release v0.9.1-honda-validation - 2026-07-02

## Summary

This release records the validated state after the TrainingComboTrials validator split and the EHonda delayed absorb validation fixes.

The release is intended as a stable checkpoint. EHonda 6928 / 7018 finish-state issues, sun / sumo-spirit resources, SA3 / CA requirements, start-state recording, requirements CSV cleanup, and Marisa SA3 final-step handling are intentionally deferred to later branches.

## Main Fixes

- Split the combo validator into focused modules:
  - `DebugTrace.lua`
  - `ActionMatcher.lua`
  - `CharacterRules.lua`
  - `Validator.lua` helper functions
- Added Sagat same-action auto-advance support.
- Added Manon `unique_resources` medal record / restore support.
- Added EHonda action id absorb handling.
- Added EHonda recent and current absorb confirmation.
- Fixed EHonda delayed absorb confirmation through `pending_current_absorb` and `recent_absorb_chain`.
- Narrowed `reset_first_step_dr_parry` behavior to avoid broader validator side effects.

## EHonda 6559 CA Note

`EHonda_COMBO_PARRY_6559_D5_SA3` is a CA-condition combo.

Observed behavior:

- Normal SA3: actual action id `1215`, combo count `11`.
- CA / low health: actual action id `1221`, combo count `12`.
- Original JSON step 15 keeps `id=1221`, `expected_combo=12`, and `damage_at_step=6559`.

Therefore, failing this combo under normal HP / normal SA3 is not a release blocker. The valid release baseline is the CA / low-health condition.
